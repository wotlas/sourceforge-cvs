Overview

      Wotlas Graphics is a flexible 2D Graphics engine you can use in applications that need 2D side-scrolling display. It mainly works around a GraphicsDirector that displays Drawable objects and uses a WindowPolicy to scroll the game screen in an appropriate manner.

      Drawable is the mother class of all the graphical objects. We provide a full set of graphical objects for our 2D engine : TextDrawable (for text display with any font you want), MotionlessSprite (to handle motionless images), Sprite (to handle images that can have motion), MultiRegionImage (to handle huge images), AuraEffect (to display a selection aura on sprites), and much more... Of course you can develop your own Drawable objects very easily.

      To load/unload images efficiently and reduce memory costs we use an ImageLibrary which is a simple image database. Our ImageLibrary can load images from disk when the application starts and/or just in time when the images are needed. The structure of the library is a tree composed of directories and images. Each directory can contain images and/or sub-directories. You can indicate directories that must have mutually exclusive sub-directories (especially useful for huge images you don't want to be in memory at the same time). The ImageLibrary can access to various image sources via an interface named ImageResourceLocator. By defining your own ImageResourceLocator you can load images from JAR files or from a network.

      Our 2D graphics engine works with Java Swing as its GraphicsDirector extends the JPanel class. But of course, because Wotlas is a GNU open-source project, you can change that easily.

We recently moved our API to support the new VolatileImage feature of Java 1.4. If you choose to not use it the API remains Java 1.3 compliant.

Finally we also added a pop-up menu system that you can use easily. Just create a Menu2DManager, define a set of SimpleMenu2D, link them and add the root menu to the Menu2DManager. Now when you catch a right mouse click on the screen you can display the right menu and listen to item being clicked.

 

API & Source

      The Wotlas Graphics API Javadoc is available here. The source is available here ( local directory source/wotlas/libs/graphics2d ) and a JAR file of its compiled classes here. It is composed of five packages :

- wotlas.libs.graphics2d : the main classes and interfaces ( GraphicsDirector, ImageLibrary, Drawable, etc. ).

- wotlas.libs.graphics2d.drawable : Drawable objects you can use ( Sprite, TextDrawable, etc. ).

- wotlas.libs.graphics2d.policy : different scrolling policies.

- wotlas.libs.graphics2d.filter : dynamic image filters. For now, the only filters available are : a color filter ( to change one or more color(s) of a sprite ) and a brightness filter.

- wotlas.libs.graphics2d.filter.color : colors for the ColorImageFilter.

- wotlas.libs.graphics2d.menu : pop-up menu defintions, see the Menu2DManager and SimpleMenu2D classes.

        If you want to rebuild the 'graphics2d.jar' file, you'll find a 'build' script in the 'build/' directory.

 

Demo

      A small demo of our API can be found here. The source is available here. Use the 'launch.bat' or 'launch.sh' script to start the demo. On Windows systems you can also double-click on 'demo.jar'.

      Use the arrows to move your character and click on other women ( characters of our Wotlas project) to display some text and a selection aura. There is a command in the 'demo/build' directory if you want to rebuild the 'demo.jar' file.
