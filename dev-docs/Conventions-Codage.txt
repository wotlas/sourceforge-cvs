2 - Conventions de nommage A RESPECTER:

    Les constantes :  TOUT_EN_MAJUSCULE

    Les variables  :  tout_en_minuscule  ou  toutenminuscule ou minusculePuisDebutMajuscule

    Les méthodes   :  minusculePuisDebutMajuscule()

    Les classes    :  TousLesDebutsEnMajuscule

   Contrairement aux exemples que je viens de donner les noms de variables
   doivent être en anglais.


3 - Les Commentaires sont OBLIGATOIRES, cela dit ne tombez pas dans l'excès.
    Pas de français... ils doivent être en english.

5 - Une fois le codage des classes terminées, n'hésitez pas à créer une petite doc contenant
    qq cas d'utilisation de votre travail. Par exemple après avoir écrit la partie API réseau
    du projet, je pourrais créer une petite doc dans /cvsroot/wotlas/dev-docs du style:

    LECON N°1: Comment envoyer un message au serveur ?

    Pour envoyer un message, il y a quatre étapes :

        // 1 - Récupération d'un lien vers la classe qui gère les envois de données
               NetSender net_sender = ...récupération

        // 2 - Récupération d'un handle sur le message que l'on veut envoyer
        //     Les messages sont classés par "catégorie"/"type du message dans la catégorie"
               MessageHelloWorld msg_hello = new MessageHelloWorld( "Bonjour Tout le Monde" );

        // 3 - demande d'envoi...
               net_sender.send( msg_hello );

        // 4 - nettoyage éventuel
               msg_hello = null;

6 - Pour chacun de vos packages vous devrez aussi créer un fichier package.html qui contiendra un petit descriptif de votre package. Jettez un coup d'oeil à /CVSROOT/wotlas/wotlassrc/wotlas/libs/net/package.html pour un exemple.
